# AI-агент для поиска и суммаризации научных статей (arXiv.org)

Данный репозиторий содержит исходный код проекта по созданию вопросно-ответной системы и системы суммаризации на базе больших языковых моделей (LLM). Проект автоматизирует процесс поиска релевантных научных публикаций в базе arXiv и формирует структурированные обзоры длинных текстов, решая проблему ограничения контекстного окна LLM.

## Архитектура системы

Управление потоком выполнения реализовано в виде конечного автомата (графа) с помощью библиотеки **LangGraph**. Граф состояний включает следующие узлы:

1. **Classifier (Классификатор)** — анализирует исходный запрос пользователя и определяет требуемое действие (намерение): быстрый ответ на вопрос (QA) или полный обзор статьи (Summarization).
2. **Rewriter (Перезапись запроса)** — преобразует пользовательский запрос в оптимизированный поисковый запрос для векторной базы данных.
3. **Retriever (Поиск)** — выполняет поиск релевантных документов в векторной базе LanceDB.
4. **Условный переход (Conditional Edge):**
   * Переход в узел **QA**, если требуется точечный ответ по контексту найденных документов.
   * Переход в узел **Summarizer**, если требуется детальный обзор статьи (с извлечением полного текста из PostgreSQL).

## Ключевые технические решения

### 1. Map-Reduce с контекстным перекрытием (ArticleProcessor)
Научные статьи часто превышают лимит токенов модели. Для решения этой проблемы реализован кастомный алгоритм обработки текста:
* **Слияние секций:** Слишком короткие подразделы статьи объединяются до достижения заданного порога токенов (`min_tokens`), чтобы оптимизировать количество вызовов к API модели.
* **Перекрытие (Overlaps):** При разбиении на чанки к каждому фрагменту добавляется контекст из предыдущей (`past_overlap`) и следующей (`future_overlap`) секций. Это предотвращает потерю смысла на границах чанков во время фазы Map.
* **Генерация:** Этап Map независимо суммаризирует каждый чанк, после чего этап Reduce собирает их в финальный связный отчет.

### 2. Отказоустойчивое управление промптами
Системные промпты вынесены в облачный сервис **LangSmith** для удобного версионирования. Реализован механизм Fallback: если сервис недоступен или отсутствует подключение к сети, система автоматически переключается на использование локальных конфигурационных файлов (`prompts.yaml`), что обеспечивает стабильность работы пайплайна.

### 3. Инфраструктура
* **Инференс LLM:** Использован фреймворк **vLLM** для ускоренной параллельной генерации токенов.
* **Базы данных:** Векторная база **LanceDB** для хранения эмбеддингов (`sentence-transformers`) и семантического поиска; **PostgreSQL** для хранения полных текстов статей, их структур и метаданных.

## Структура репозитория

* `agent.py` — реализация графа выполнения LangGraph и основного класса `ArxivAgent`.
* `processing.py` — логика предобработки текстов (разбиение, склейка, расчет токенов и перекрытий).
* `summarization.py` — реализация пайплайна Map-Reduce и интеграция с провайдером vLLM.
* `prompts.yaml` — локальные конфигурации системных промптов (запасной вариант).

## Стек технологий
* Python 3.10+
* LangGraph / LangChain
* vLLM, HuggingFace Transformers
* LanceDB, PostgreSQL
* LangSmith (трассировка и хранение промптов)

## Запуск проекта

Проект рассчитан на выполнение в среде с GPU (например, Kaggle Notebooks). 

1. Клонирование репозитория:
```bash
git clone https://github.com/ВАШ_НИК/Summarization-scientific-articles.git
```

2. Установка зависимостей:
```bash
pip install vllm transformers langgraph langsmith langchain langchainhub lancedb sentence-transformers psycopg2
```

3. Инициализация и вызов агента:
```python
# Настройка доступов к LangSmith и БД
# ... (определение переменных окружения)

# Инициализация компонентов
agent = ArxivAgent(
    llm_provider=provider,
    retriever=retriever,
    sum_pipeline=sum_pipe,
    processor=processor,
    embed_model=retrieval_model,
    db_params=db_params,
    tokenizer=tokenizer,
    prompts=hub_prompts_config, 
    local_prompts=local_prompts_yaml,
    use_hub=True # False для отключения LangSmith
)

# Запуск
response = agent.invoke("Сделай подробный обзор статьи про механизм Attention")
print(response['final_answer'])
```
